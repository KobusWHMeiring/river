# PRD: Data Export for Reporting (Excel)

## 1. Problem Statement
Managers need to extract data from the system for donor reporting and offline analysis. Currently, data is locked within the web interface, making it difficult to include in external documents or perform complex cross-referencing.

## 2. Strategic Goal
Provide a comprehensive, professional Excel export that summarizes project-wide impact (Overview) and provides granular details organized by river section.

## 3. Proposed Scope
- **Format:** `.xlsx` (Excel) using the `openpyxl` library.
- **Access Points:** "Export to Excel" buttons on the Global Dashboard and Section List.
- **Workbook Structure:**
    - **Sheet 1: Project Overview**: A high-level summary of all metrics (Bags collected, plants planted, weeding sessions) and species breakdowns.
    - **Sheets 2-N: [Section Name]**: Individual sheets for each river section containing its specific Visit Logs, Tasks, and aggregated metrics.
- **Data Coverage:** v1 will export all historical data (no date filters yet).

## 4. Technical Constraints
- **Library:** Must use `openpyxl` (lightweight, native Excel support).
- **Performance:** For v1, the export will be generated synchronously. If the dataset grows significantly, we may need to move to background tasks (not in scope for v1).
- **Styling:** Use basic Excel styling (bold headers, alternating row colors) to ensure the report is "donor-ready."

## 5. Developer Context (Current State)

### What Already Exists
- **Aggregation Logic:** `GlobalDashboardView` (`core/views.py`) already contains the logic for calculating total bags, plants, and species breakdowns. This can be abstracted into a utility or reused.
- **Model Relationships:** `VisitLog` and `Task` are both linked to `Section`. `Metric` is linked to `VisitLog`.
- **UI Patterns:** Standard "Action Buttons" are used throughout the app (e.g., `primary` and `secondary` styled buttons).

### What Needs to Be Created
- **Dependency:** Add `openpyxl` to `requirements.txt`.
- **DataExportView:** A new class-based view in `core/views.py` that handles the Excel generation and returns a `FileResponse`.
- **URL Pattern:** `core/export/` path in `core/urls.py`.
- **UI Elements:** Buttons in `dashboard.html` and `section_list.html`.

# User Stories (Generated by Technical PO)

## Story 1: Excel Export Infrastructure
**Value Proposition:** As a Developer, I want to add the necessary libraries and view structure so that I can begin generating Excel files.

**Technical Implementation Path:**
- Target File: `requirements.txt` (Add `openpyxl`)
- Target File: `core/views.py` (Create `DataExportView(LoginRequiredMixin, View)`)
- Target File: `core/urls.py` (Add `path('export/', views.DataExportView.as_view(), name='data_export')`)

**Acceptance Criteria (AC):**
- [ ] View returns an empty `.xlsx` file with a proper filename: `Liesbeek_River_Report_[YYYY-MM-DD].xlsx`.
- [ ] Content-Type is set to `application/vnd.openxmlformats-officedocument.spreadsheetml.sheet`.

**The Test Plan:**
- **Unit Test:** Verify the URL resolves and returns a 200 status with the correct content-type header.

---

## Story 2: Global Overview Sheet
**Value Proposition:** As a Manager, I want a summary sheet at the start of the workbook so that I can quickly see total impact metrics for donor reports.

**Technical Implementation Path:**
- Target File: `core/views.py` (In `DataExportView.get`)
  - Create the "Overview" sheet.
  - Populate with: Total Bags (General/Recyclable), Total Plants, Total Weeding Sessions.
  - Include "Top 5 Species" tables for both Planting and Weeding.

**Acceptance Criteria (AC):**
- [ ] Overview sheet is the first sheet in the workbook.
- [ ] Data matches the numbers shown on the Global Dashboard.
- [ ] Headers are bolded for readability.

**The Test Plan:**
- **Integration Test:** Compare export totals against the Dashboard UI values using a test database with known values.

---

## Story 3: Per-Section Detail Sheets
**Value Proposition:** As a Manager, I want granular data organized by section so that I can review the history of specific parts of the river independently.

**Technical Implementation Path:**
- Target File: `core/views.py` (Loop through `Section.objects.all()`)
  - Create a new sheet for each section (title limited to 31 chars per Excel spec).
  - List all `VisitLog` entries (Date, Notes, Summary of Metrics).
  - List all `Task` entries (Date, Instructions, Completed Status).

**Acceptance Criteria (AC):**
- [ ] Every active section has its own sheet.
- [ ] Sheet names are clean (e.g., "Section A - Upper").
- [ ] Lists are sorted by date (newest first).

**The Test Plan:**
- **Edge Case:** Handle sections with very long names (truncate to 31 chars).
- **Edge Case:** Handle sections with zero logs or tasks (show "No activity recorded").

---

## Story 4: UI Integration
**Value Proposition:** As a User, I want to trigger the export easily from the main data views.

**Technical Implementation Path:**
- Target File: `core/templates/core/dashboard.html`
- Target File: `core/templates/core/section_list.html`
  - Add `<a>` tag styled as a button next to the "Add" buttons.
  - Use `download` icon and text "Export to Excel".

**Acceptance Criteria (AC):**
- [ ] Buttons are visible and accessible.
- [ ] Clicking the button triggers the download immediately without page reload.

**The Test Plan:**
- **Manual Test:** Click button on both pages and verify the resulting file opens correctly in Excel/Google Sheets.
