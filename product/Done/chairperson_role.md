# PRD: Chairperson Assignee Type & Planner Integration

## 1. Problem Statement
The current system tracks "Team" (Field Workers) and "Manager" (Project Oversight) tasks. There is a requirement for a third high-level category: **Chairperson**. This role handles strategic, high-level activities (e.g., donor relations, board meetings) that need to be planned and visualized separately on the planners to avoid cluttering operational views.

## 2. Strategic Goal
Introduce the "Chairperson" actor into the system to support strategic planning. This feature must be implemented first as it expands the core "Assignee" dimension used by all future features (Multi-Day, To-Do, etc.).

## 3. Proposed Scope
- **Data Model:** Update `ASSIGNEE_TYPE_CHOICES` in `Task` and `TaskTemplate` models.
- **Weekly Planner:** A new horizontal row titled "Chairperson Strategy" with a distinct soft pastel visual style.
- **Monthly Planner:** A third grouping within day cells for Chairperson tasks.
- **Task Creation:** Full integration into the "New Task" modal, including template filtering for the Chairperson role.
- **Daily Agenda:** Grouping and ordering to support the new role.

## 4. UX/UI Concept
- **Visual Style:** Soft Pastel Rose/Pink (`bg-rose-50`, `text-rose-600`, `border-rose-100`).
- **Icon:** `account_balance` or `workspace_premium` (Material Symbols).
- **Layout:** In the Weekly Planner, the Chairperson row will sit between Manager and Team (or as the top-most row for strategic priority).

## 5. Risks
- **UI Density:** Adding a 3rd row to the Monthly Planner's day cells might cause more frequent "overflow" indicators. *Mitigation:* Adjust the default slice to show 1-2 tasks per type before showing the "+X more" link.

# Technical Blueprint: Chairperson Actor Integration

## 1. Database / Models
- **Task/TaskTemplate (`core/models.py`):**
    - Update `ASSIGNEE_TYPE_CHOICES`:
      ```python
      ASSIGNEE_TYPE_CHOICES = [
          ('team', 'Team'),
          ('manager', 'Manager'),
          ('chairperson', 'Chairperson'),
      ]
      ```

## 2. Queries & ORM Strategy
- **Planner Fetching:** Ensure `WeeklyPlannerView` and `MonthlyPlannerView` use `select_related('section')` and correctly filter templates by the new `chairperson` type.

## 3. Views / API Endpoints
- **core/views.py:** 
    - Update `WeeklyPlannerView.get_context_data` to include `chairperson_task_templates`.
    - Update `MonthlyPlannerView.get_context_data` to include `chairperson_task_templates`.
- **core/forms.py:** 
    - Ensure `TaskForm` handles the new choice (it should automatically if using model choices, but check explicit widget overrides).

## 4. Step-by-Step Implementation Checklist
1. **Model:** Update `ASSIGNEE_TYPE_CHOICES` and generate migration.
2. **View Logic:** Update `WeeklyPlannerView` and `MonthlyPlannerView` to pass `chairperson_task_templates` to context.
3. **Weekly Template:** Duplicate the "Manager Tasks" section in `weekly_planner.html`, rename to "Chairperson Strategy", and update icons/colors.
4. **Monthly Template:** Update day-cell task filtering in `monthly_planner.html` to include a `chairperson_tasks` group.
5. **Modal JS:** Update the template-switching logic in `weekly_planner.html` and `monthly_planner.html` to support the `chairperson` type.
